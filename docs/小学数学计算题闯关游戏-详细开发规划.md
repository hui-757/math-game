# 小学数学计算题闯关游戏 — 详细开发规划

> 基于甘肃省庆阳市宁县早胜小学 曹鹏英 老师的需求文档
> 版本：v1.0 | 日期：2025-07-06

---

## 一、项目定位与目标

### 1.1 项目定位
打造一款面向小学1-6年级学生的**数学计算题闯关游戏化学习平台**，将枯燥的计算练习转化为有趣的关卡挑战，通过视觉化的进度反馈（如长城攀登、取经路线等主题）激发学生学习兴趣，同时为教师提供精准的学情数据。

### 1.2 核心目标
- **学生端**：让计算练习变得有趣，提升练习积极性和成就感
- **教师端**：提供学情数据分析，辅助精准教学
- **内容端**：覆盖1-6年级12册全部计算内容，系统化整合

### 2.2 核心功能模块

#### 模块一：内容体系管理（基础层）

| 层级 | 说明 | 示例 |
|------|------|------|
| **学段** | 小学 | 小学 |
| **年级** | 1-6年级 | 三年级 |
| **册次** | 上册/下册 | 下册 |
| **单元** | 教材单元 | 第2单元：除数是一位数的除法 |
| **关卡** | 单元内的细分挑战 | 关卡1：口算热身；关卡2：笔算挑战；关卡3：脱式计算；关卡4：综合应用 |
| **题目** | 最小内容单元 | 具体计算题 |

**内容覆盖计划（12册 × 约8-10单元/册 × 约4-5关/单元）**
- 预计总关卡数：约 **400-600关**
- 预计总题数：约 **3000-5000题**

#### 模块二：关卡与地图系统（体验层）

**主题方案对比（推荐二选一或支持切换）**：

| 主题 | 视觉风格 | 关卡节点设计 | 适合氛围 |
|------|---------|-------------|---------|
| **长城关卡** | 中国风、雄伟、攀登感 | 烽火台作为关卡节点，通关后角色移动到下一个烽火台 | 激励、挑战、坚持 |
| **西游记取经** | 古典神话、趣味、冒险 | 取经路线图（长安→西天），每过一关到达一个地点（流沙河、火焰山等） | 冒险、趣味、故事性 |
| **太空探险** | 科幻、未来感 | 星球/空间站作为节点 | 探索、科技感 |

**推荐**：优先实现**长城关卡**（符合中国文化认同），同时预留**西游记取经**作为第二个主题皮肤。

**地图交互设计**：
- 大地图展示当前册次的所有关卡节点
- 已通关节点：高亮、插旗、可点击回顾
- 当前关卡：闪烁、角色站立、可点击进入
- 未解锁节点：灰色、锁图标、无法点击
- 支持缩放、拖拽浏览大地图

#### 模块三：答题与反馈系统（核心层）

**答题流程**：
```
进入关卡 → 展示题目（1题/次或5题/组） → 学生输入答案 → 即时判断 → 
正确：动画+音效+积分 → 错误：提示+鼓励语 → 记录结果 → 判断是否通关
```

**通关规则**：
- 每关设置 **5-10道题**
- 正确率 **≥80%**（可配置）视为通关
- 未通关可重复挑战，记录最佳成绩

**反馈设计（非简单音效）**：

| 类型 | 正确反馈 | 错误反馈 |
|------|---------|---------|
| 视觉 | 烟花特效、角色跳跃、金币飞溅、关卡节点点亮动画 | 角色摇头、 gently 提示框、正确答案闪烁 |
| 听觉 | 欢快的通关音乐、掌声、金币声 | 温和的鼓励语音："再想一想，你可以的！" |
| 文案 | "太棒了！你又向前迈了一步！"、"小小数学家！" | "没关系，再试一次！"、"加油，你离成功只差一步！" |
| 动画 | 角色从当前节点移动到下一个节点（长城：攀登到下一个烽火台） | 停留在原地，显示错题解析按钮 |

#### 模块四：学情数据系统（教师层）

**学生个人数据**：
- 闯关进度（总关卡数、已通关数、通关率）
- 正确率趋势（日/周/月统计）
- 易错题型TOP10（如：退位减法、小数除法）
- 平均用时/每题
- 连续打卡天数

**班级学情报告**：
- 班级整体通关率热力图
- 各单元平均正确率对比
- 班级易错题排行榜
- 进步之星/计算小能手榜单
- 未完成任务学生提醒

**数据导出**：
- 支持导出Excel/CSV格式的学情报告
- 支持打印纸质版反馈单

#### 模块五：用户与权限系统

| 角色 | 权限 |
|------|------|
| **学生** | 选择年级册次、闯关答题、查看个人成绩 |
| **教师** | 创建班级、添加学生、查看学情、自定义题目 |
| **管理员** | 内容管理、用户管理、系统配置、数据统计 |

## 三、内容规划（1-6年级计算内容）

### 3.1 内容组织原则

1. **与教材同步**：按人教版/北师大版等主流教材的单元结构组织
2. **螺旋上升**：低年级内容在高年级以综合形式复现
3. **难度递进**：每单元内从基础口算 → 笔算 → 脱式 → 综合应用
4. **题型多样**：口算、估算、竖式计算、脱式计算、简便运算、解方程等

### 3.2 各年级计算内容清单

#### 一年级（上册）
- 10以内数的认识和加减法
- 20以内进位加法
- 认识钟表（不涉及复杂计算）
- **计算重点**：10以内加减法、20以内进位加法

#### 一年级（下册）
- 20以内退位减法
- 100以内数的认识
- 100以内加减法（整十数加减、两位数加减整十数/一位数）
- **计算重点**：20以内退位减法、100以内简单加减法

#### 二年级（上册）
- 100以内加减法（进位、退位）
- 表内乘法（2-9）
- **计算重点**：100以内加减法竖式、表内乘法

#### 二年级（下册）
- 表内除法（2-9）
- 混合运算（同级运算）
- 有余数的除法
- **计算重点**：表内除法、有余数除法、简单混合运算

#### 三年级（上册）
- 万以内加减法
- 多位数乘一位数
- 分数的初步认识（简单分数比较/计算）
- **计算重点**：万以内加减法、多位数乘一位数

#### 三年级（下册）【需求文档重点】
- 除数是一位数的除法（笔算）
- 两位数乘两位数
- 三位数加减两位数/三位数
- 脱式计算
- 简单的小数加减法
- **计算重点**：笔算除法（除数一位数）、两位数乘两位数、脱式计算

#### 四年级（上册）
- 大数的认识（不涉及计算）
- 三位数乘两位数
- 除数是两位数的除法
- **计算重点**：三位数乘两位数、除数是两位数的除法

#### 四年级（下册）
- 四则运算（含括号、运算顺序）
- 运算定律（交换律、结合律、分配律）简便运算
- 小数的意义和性质
- 小数加减法
- **计算重点**：四则混合运算、运算定律简便运算、小数加减法

#### 五年级（上册）
- 小数乘法
- 小数除法
- **计算重点**：小数乘除法

#### 五年级（下册）【需求文档重点】
- 简易方程（解方程）
- 分数的加减法
- **计算重点**：解方程、分数加减法

#### 六年级（上册）
- 分数乘法
- 分数除法
- 比和比例（不涉及复杂计算）
- **计算重点**：分数乘除法、分数四则混合运算

#### 六年级（下册）【需求文档重点：综合计算】
- 负数
- 百分数（二）（折扣、成数、税率、利率）
- 圆柱圆锥（涉及π的计算）
- 整理和复习：数的运算综合
- **计算重点**：整数/小数/分数/百分数综合计算、解方程、脱式计算、简便运算

### 3.3 题目生成策略

**方案A：预置题库**（推荐）
- 按每个关卡预置20-30道题，随机抽取5-10道
- 优点：质量可控，有解析，反馈精准
- 缺点：初期工作量较大

**方案B：程序生成**
- 根据规则实时生成题目（如：两位数×两位数，随机生成）
- 优点：题量无限，永不重复
- 缺点：需要校验生成质量，解析需自动生成

**推荐组合**：核心关卡用预置题库，练习模式用程序生成作为补充。

## 四、技术架构规划

### 4.1 技术选型方案

考虑到小学场景（学校机房、家庭平板/手机、无需复杂安装），推荐以下方案：

| 方案 | 前端 | 后端 | 数据库 | 部署 |
|------|------|------|--------|------|
| **方案A：Web应用**（推荐） | Vue 3 + Canvas/SVG地图 | Python Flask / Node.js | PostgreSQL / MySQL | 云服务器 / 学校内网 |
| **方案B：小程序** | 微信小程序 | 云开发 | 云数据库 | 微信生态 |
| **方案C：单机桌面** | Electron + Vue | 本地JSON/SQLite | SQLite | 本地安装 |

**推荐方案A（Web应用）**，理由：
- 跨平台：电脑、平板、手机均可访问
- 免安装：浏览器打开即用，适合学校机房环境
- 易维护：统一服务器更新，无需每台设备维护
- 数据集中：学生数据统一存储，教师随时查看
- 可扩展：后续可扩展为小程序、APP等

## 七、关键设计决策

### 7.1 题目输入方式

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 键盘输入 | 适合PC、准确 | 移动端不便 | 主要方案 |
| 手写板 | 保留书写过程 | 识别精度、开发复杂 | 可选增值功能 |
| 选择题 | 移动端友好 | 可能猜测 | 低年级辅助 |
| 语音输入 | 新颖 | 环境干扰、识别精度 | 暂不实现 |

**决策**：以**键盘输入**为主，低年级可增加**选项选择**作为辅助模式。

### 7.2 关卡解锁机制

| 方案 | 说明 | 建议 |
|------|------|------|
| 线性解锁 | 必须按顺序一关一关过 | 过于严格，可能挫败 |
| 单元内自由 | 单元内关卡自由挑战，但需通关前一单元解锁下一单元 | **推荐**：既有自由度又有进阶感 |
| 全自由 | 所有关卡随意挑战 | 缺乏目标感 |

**决策**：**单元内自由挑战，单元间线性解锁**。学生可以在当前单元内任选关卡，但需通关本单元大部分关卡才能解锁下一单元。

### 7.3 积分/奖励体系

- **星级评价**：每关1-3星（根据正确率和用时）
- **连击奖励**：连续正确获得额外积分
- **每日任务**：完成5关获得"今日小勇士"徽章
- **成就系统**：
  - "计算新秀"：通关10关
  - "长城攀登者"：通关一册全部关卡
  - "学期达人"：通关一学期（上下册）
  - "数学小博士"：通关全部年级
