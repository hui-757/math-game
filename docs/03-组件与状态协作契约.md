# 组件与状态协作契约

> 版本：v1.0 | 对应代码版本：`main` 分支 2025-07-03
> 适用范围：math-game 前端 Vue 3 项目

## 3.1 组件接口规范（Props / Events / Expose）

所有页面级组件通过 Vue Router 的 `props: true` 接收路由参数，组件间不直接传递数据，全部通过 Pinia Store 或路由参数解耦。

### 3.1.1 SelectView（年级选择页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | 无 | — | 纯展示组件，无外部参数 |
| Events | 无 | — | 通过 `router.push()` 导航 |
| 路由输出 | `/semester/:grade` | — | 点击年级卡片跳转 |

### 3.1.2 SemesterView（册次选择页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | `grade` | `string` | 路由参数，年级 1~6 |
| 路由输出 | `/unit/:grade/:semester` | — |  semester = 1（上册）或 2（下册） |

### 3.1.3 UnitView（单元列表页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | `grade` | `string` | 年级 1~6 |
| Props | `semester` | `string` | 学期 1/2 |
| 内部依赖 | `progressStore` | Store | 读取关卡完成状态 |
| 内部依赖 | `fetchQuestions()` | API | 查询数据库哪些 unit 有题目 |
| 路由输出 | `/play/:levelId` | — | levelId 格式：`{grade}-{semester}-{unit}-1` |

### 3.1.4 PlayView（答题页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | `levelId` | `string` | 格式：`{grade}-{semester}-{unit}-{level}`，例 `"1-2-3-1"` |
| 内部依赖 | `gameStore` | Store | 提交结果后写入 `lastResult` |
| 内部依赖 | `progressStore` | Store | 提交结果后写入本地记录 |
| 内部依赖 | `authStore` | Store | 登录用户时同步到云端 |
| 路由输出 | `/result` | — | 提交后跳转结果页 |

**`levelId` 解析规则**：
```js
const [grade, semester, unit, level] = levelId.split('-').map(Number)
// 例："1-2-3-1" → grade=1, semester=2, unit=3, level=1
```

### 3.1.5 ResultView（结果页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | 无 | — | 纯读取 `gameStore.lastResult` |
| 内部依赖 | `gameStore.lastResult` | Store | `{ levelId, correct, total, time, mistakes, score }` |
| 路由输出 | `/play/:levelId` | — | "再来一次" |
| 路由输出 | `/unit/:grade/:semester` | — | "返回单元列表" |
| 路由输出 | `/select` | — | "回首页" |

### 3.1.6 TeacherView（教师/个人中心页）

| 接口类型 | 名称 | 类型 | 说明 |
|----------|------|------|------|
| Props | 无 | — | |
| 内部依赖 | `authStore` | Store | 读取/设置学生信息 |
| 内部依赖 | `progressStore` | Store | 读取本地记录列表 |

## 3.2 Pinia Store 状态树与 Action 签名

### 3.2.1 `useAuthStore`（身份认证）

```js
// 状态树
const student   = ref(null)      // { id: string, name: string } | null
const classInfo = ref(null)      // 预留，当前未使用

// Actions
function setStudent(data)     // 写入 student + localStorage
function setClassInfo(data)   // 写入 classInfo + localStorage
function loadFromStorage()    // 从 localStorage 恢复 student/classInfo
function logout()             // 清空 student/classInfo + 移除 localStorage
```

### 3.2.2 `useGameStore`（临时游戏状态）

```js
// 状态树
const currentQuestions = ref([])   // 当前关卡题目数组
const currentIndex     = ref(0)    // 当前题号
const userAnswers      = ref({})   // { blankId: value }
const startTime        = ref(0)    // 开始时间戳（ms）
const elapsedTime      = ref(0)    // 用时（秒）
const levelId          = ref('')   // 当前关卡ID
const lastResult       = ref(null) // 最近一次闯关结果，供 ResultView 读取

// Actions
function initGame(questions, level)   // 初始化游戏状态
function setAnswer(blankId, value)    // 记录单题答案
function nextQuestion()               // 题号+1
function finishGame()                   // 计算 elapsedTime
function setResult(result)              // 写入 lastResult
function clearGame()                    // 清空所有游戏状态
```

**注意**：`initGame` / `setAnswer` / `nextQuestion` / `finishGame` 当前在 PlayView 中**未被使用**（PlayView 使用本地 `ref` 管理状态）。预留供未来重构统一调用。

### 3.2.3 `useProgressStore`（持久化进度）

```js
// 状态树
const records = ref([])
// 单条记录格式：
// {
//   level_id: string,      // 例 "1-2-3-1"
//   correct_count: number,
//   total_count: number,
//   time_seconds: number,
//   mistakes: number,
//   timestamp: number        // Date.now()
// }

// Actions
async function load(studentId)         // 从 Supabase 拉取远程进度
function loadFromStorage()             // 从 localStorage 恢复
function saveToStorage()               // 写入 localStorage
function addRecord(record)              // 去重添加（同 level_id 覆盖）
function isLevelCompleted(levelId)      // 是否已完成
function isLevelUnlocked(grade, semester, unit, level)  // 关卡是否解锁
function getLevelStars(levelId)         // 返回 0/1/2/3 星
```

**解锁规则**：
- `level === 1`：始终解锁
- 其他：前一关 `level - 1` 必须已完成
- 当前每个 unit 只有 1 个 level（固定为 1）

## 3.3 路由守卫与状态恢复策略

### 3.3.1 当前状态：无全局路由守卫

项目当前**未配置 `beforeEach` / `beforeEnter` 路由守卫**，所有页面直接渲染。

### 3.3.2 各页面状态恢复策略

| 页面 | 恢复时机 | 恢复内容 | 降级行为 |
|------|----------|----------|----------|
| SelectView | 无 | 无 | — |
| SemesterView | 无 | 无 | — |
| UnitView | `onMounted` | `progressStore.loadFromStorage()` + 查询数据库 | 无进度时全部显示"可闯关" |
| PlayView | `onMounted` | `authStore.loadFromStorage()` + 拉取题目 | 无题目时显示错误提示 |
| ResultView | 计算属性 | `gameStore.lastResult` | 无结果时显示"暂无闯关记录" |
| TeacherView | `onMounted` | `authStore.loadFromStorage()` + `progressStore.loadFromStorage()` | 无登录时显示登录表单 |

### 3.3.3 页面刷新/后退时的数据持久化

- **PlayView 刷新**：`allAnswers` 全部丢失，需重新答题。当前未实现 mid-game 保存。
- **ResultView 刷新**：`lastResult` 为 `null`（内存引用），显示空状态。建议未来使用 `sessionStorage` 缓存。
- **进度记录**：`progressStore` 通过 `localStorage` 自动持久化，刷新不丢失。

## 3.4 错误码与降级方案

### 3.4.1 错误分类与处理

| 错误场景 | 触发位置 | 错误表现 | 降级方案 |
|----------|----------|----------|----------|
| **网络中断**（Supabase 连接失败） | `fetchQuestions()` | `error` 被抛出 | `catch` 后显示 `"加载失败: ..."`，提供"返回"按钮 |
| **题目缺失**（数据库无该 unit 题目） | PlayView `onMounted` | `questions.length === 0` | 显示 `"该关卡暂无题目"`，可返回单元列表 |
| **进度同步失败**（云端写入失败） | PlayView `submitAll` | `submitProgress()` 抛错 | `console.warn` 静默失败，本地记录已保存 |
| **JSON 解析失败**（content 字段损坏） | PlayView `parsedContent` | `JSON.parse` 抛错 | `catch` 返回 `{ lines: [[{type:'text', text: q.content}]] }` 兜底 |
| **localStorage 损坏** | `loadFromStorage` | `JSON.parse` 抛错 | `try/catch` 后设为空数组/空对象 |
| **未登录访问教师页** | TeacherView | 无 | 正常显示登录表单，不强制跳转 |

### 3.4.2 错误码约定（预留）

当前代码未定义统一错误码，建议后续增加：

```js
const ERR_CODES = {
  NET_TIMEOUT: 'E001',      // 网络超时
  DB_NO_DATA: 'E002',       // 数据库无数据
  JSON_INVALID: 'E003',     // 题目格式损坏
  AUTH_REQUIRED: 'E004',    // 需要登录（预留）
  STORAGE_CORRUPT: 'E005',  // 本地存储损坏
}
```

## 3.5 本地缓存 Schema（localStorage Key 约定）

### 3.5.1 当前使用的 Key

| Key | 数据类型 | 写入方 | 读取方 | 生命周期 |
|-----|----------|--------|--------|----------|
| `math-game-student` | `JSON` | `authStore.setStudent()` | `authStore.loadFromStorage()` | 持久，logout 时删除 |
| `math-game-class` | `JSON` | `authStore.setClassInfo()` | `authStore.loadFromStorage()` | 持久，logout 时删除 |

// math-game-progress

  {
    "level_id": "1-2-3-1",
    "correct_count": 8,
    "total_count": 10,
    "time_seconds": 145,
    "mistakes": 2,

  }

### 3.5.3 缓存策略建议

| 策略 | 状态 | 说明 |
|------|------|------|
| 题目内容缓存 | ❌ 未实现 | 建议 `sessionStorage` 缓存当前关卡题目，防刷新丢失 |
| 答题中答案缓存 | ❌ 未实现 | 建议 `sessionStorage` 缓存 `allAnswers`，刷新后可恢复 |
| 结果页缓存 | ❌ 未实现 | 建议 `sessionStorage` 缓存 `lastResult`，刷新后仍可见 |
| 缓存过期清理 | ❌ 未实现 | 建议每次启动时清理 7 天前的 `sessionStorage` 数据 |

### 3.6.3 使用禁区

- ❌ **禁止在组件内直接使用** `rgb()` / `hsl()` / `#rrggbbaa` 等非常量格式，保持统一 hex
- ❌ **禁止在 `<style>` 中写死** 与当前主题冲突的颜色（如紫色渐变、蓝色渐变等背景）
- ❌ **禁止在组件内使用** `!important` 覆盖颜色
- ✅ **建议**：所有颜色统一通过 CSS 变量 `:root {}` 或 `theme.css` 管理，组件只引用变量名

## 3.7 动画时序与回调约定

### 3.7.1 当前状态：无动画系统

当前项目**未实现任何动画效果**（无解锁动画、无过渡动画、无庆祝效果）。所有页面切换和状态变化均为即时渲染。

### 3.7.2 预留动画时序约定

如需后续添加动画，建议遵守以下时序：

| 动画场景 | 建议时长 | 可交互时机 | 说明 |
|----------|----------|------------|------|
| 关卡解锁动画 | 800ms | 动画结束后 200ms | 解锁图标缩放+颜色变化，完成后可点击 |
| 答题正确反馈 | 300ms | 即时 | 输入框绿色闪烁，不阻塞操作 |
| 答题错误反馈 | 500ms | 即时 | 输入框红色抖动，不阻塞操作 |
| 提交结果过渡 | 600ms | 动画结束后 | 从答题页淡出到结果页 |
| 星级展示动画 | 400ms/星 | 全部展示后 | 逐颗星星弹出，完成后显示按钮 |
| 错题本展开 | 300ms | 即时 | 手风琴展开，可点击其他区域 |

### 3.7.3 回调约定（Vue 过渡钩子）

```vue
<Transition
  @before-enter="onBeforeEnter"
  @after-enter="onAfterEnter"
  @before-leave="onBeforeLeave"
  @after-leave="onAfterLeave"
>
  <!-- 动画内容 -->
</Transition>
```

| 钩子 | 用途 | 回调命名建议 |
|------|------|--------------|
| `before-enter` | 动画开始前初始化状态 | `animBeforeEnter` |
| `after-enter` | 动画结束后解锁交互 | `animAfterEnter` |
| `before-leave` | 离开动画前保存状态 | `animBeforeLeave` |
| `after-leave` | 离开动画后清理 DOM | `animAfterLeave` |

### 3.7.4 当前代码中的 `transition` 使用

```css
/* 仅存在以下简单的 CSS transition，不涉及动画时序 */
.grade-card:hover { transition: transform 0.2s, box-shadow 0.2s; }
.blank-input { transition: border-color 0.2s; }
.btn { transition: opacity 0.2s; }
```

**结论**：当前项目为 MVP 阶段，动画系统完全未实现，所有交互为即时响应。如需增加动画效果，需先统一设计动画时序规范，再按本章节约定实施。
