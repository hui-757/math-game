## 一、数据格式契约（JSON Schema）

所有题库文件、API 返回、组件 props 都必须符合以下格式。

### 1.1 题目单条格式（Question Item）

```json
{
  "id": "q-3-2-1-1-001",
  "grade": 3,
  "semester": 2,
  "unit": 1,
  "unit_name": "除数是一位数的除法",
  "level": 1,
  "type": "vertical",
  "difficulty": 2,
  "content": {
    "lines": [
      [{"type": "text", "text": "126 ÷ 3 = "}, {"type": "blank", "id": "b1", "answer": "42"}]
    ]
  },
  "answer": "42",
  "display": "126 ÷ 3 = ?"
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 全局唯一，格式 `q-{grade}-{semester}-{unit}-{level}-{seq}` |
| `grade` | int | ✅ | 年级：1~6 |
| `semester` | int | ✅ | 学期：1=上册，2=下册 |
| `unit` | int | ✅ | 单元序号：1,2,3... |
| `unit_name` | string | ✅ | 单元名称，如"除数是一位数的除法" |
| `level` | int | ✅ | 关卡序号：1,2,3... |
| `type` | string | ✅ | 题型：`oral` / `vertical` / `step` / `equation` |
| `difficulty` | int | ✅ | 难度：1=简单，2=中等，3=困难 |
| `content` | object | ✅ | 题目结构，见下方 |
| `answer` | string | ✅ | 最终答案（用于判题） |
| `display` | string | ✅ | 纯文本展示用 |

**`content.lines` 结构：**

每行是一个数组，元素为 `{"type": "text", "text": "..."}` 或 `{"type": "blank", "id": "b1", "answer": "..."}`。

```json
// 口算示例
{
  "lines": [[
    {"type": "text", "text": "15 + 28 = "},
    {"type": "blank", "id": "b1", "answer": "43"}
  ]]
}

// 竖式示例（多位商，每空一位）
{
  "lines": [
    [{"type": "text", "text": "  3"}, {"type": "blank", "id": "q0", "answer": "4"}, {"type": "blank", "id": "q1", "answer": "2"}],
    [{"type": "text", "text": "3)126"}]
  ]
}

// 脱式示例
{
  "lines": [
    [{"type": "text", "text": "15 + 8 × 2 = 15 + "}, {"type": "blank", "id": "s1", "answer": "16"}, {"type": "text", "text": " = "}, {"type": "blank", "id": "s2", "answer": "31"}]
  ]
}

// 方程示例
{
  "lines": [[
    {"type": "text", "text": "3x + 5 = 20"},
    {"type": "text", "text": "  x = "},
    {"type": "blank", "id": "e1", "answer": "5"}
  ]]
}
```

### 1.2 题库文件命名约定

```
data/questions/
├── grade01-semester1.json   # 一年级上册
├── grade01-semester2.json   # 一年级下册
├── grade02-semester1.json
├── ...
└── grade06-semester2.json   # 六年级下册
```

**每个文件是一个数组： `[Question Item, Question Item, ...]`**

**按文件即可划分年级册次**：前端 import 或上传时，根据文件名解析 `grade` 和 `semester`。

### 1.3 关卡分组规则（前端负责）

同一文件内，按 `(unit, level)` 分组，每关默认 **10 题**：

```js
// 前端分组逻辑
const levels = groupBy(questions, q => `${q.unit}-${q.level}`)
// levels["1-1"] = [q1, q2, ..., q10]  // 第1单元第1关
// levels["1-2"] = [q11, q12, ..., q20] // 第1单元第2关
```

**约束：** 子任务交付时，每个 `unit-level` 组合必须有且仅有 10 题（最后关可少于10题）。

## 二、API 契约（Supabase 接口层）

主团队负责封装 `src/lib/api.js`，子任务不直接调 Supabase，只通过此文件。

### 2.1 封装层约定

```js
// src/lib/api.js —— 主团队维护，子任务只读

import { createClient } from '@supabase/supabase-js'
const supabase = createClient(import.meta.env.VITE_SUPABASE_URL, import.meta.env.VITE_SUPABASE_KEY)

// ========== 题目接口 ==========

/** 
 * 按年级册次拉取题目
 * @param {number} grade 年级
 * @param {number} semester 学期
 * @returns {Promise<QuestionItem[]>}
 */
export async function fetchQuestions(grade, semester) {
  const { data, error } = await supabase
    .from('questions')
    .select('*')
    .eq('grade', grade)
    .eq('semester', semester)
    .order('unit', { ascending: true })
    .order('level', { ascending: true })
  if (error) throw error
  return data
}

/** 
 * 批量导入题目（仅管理员/脚本使用）
 * @param {QuestionItem[]} questions 
 */
export async function importQuestions(questions) {
  const { error } = await supabase.from('questions').upsert(questions)
  if (error) throw error
}

// ========== 进度接口 ==========

/**
 * 获取某学生的闯关进度
 * @param {string} studentId 
 * @returns {Promise<ProgressItem[]>}
 */
export async function fetchProgress(studentId) {
  const { data } = await supabase
    .from('progress')
    .select('*')
    .eq('student_id', studentId)
  return data || []
}

/**
 * 提交关卡成绩
 * @param {object} payload 
 */
export async function submitProgress(payload) {
  const { error } = await supabase.from('progress').upsert(payload)
  if (error) throw error
}

}
```

### T6：题型组件（主团队，6h）

**输入：** T1 + T5
**输出：** 4个独立可用的题型组件

**复用基础：** `MathBlank.vue`（统一填空输入）

| 组件 | 说明 | 验收 |
|------|------|------|
| `OralQuestion.vue` | 一行文本 + 1个blank | 能输入数字，提交能取到值 |
| `VerticalQuestion.vue` | Grid布局，被除数/除数/商对齐 | 多位商每格独立输入 |
| `StepQuestion.vue` | 多行文本流，每行末尾blank | 等号对齐，输入框不溢出 |
| `EquationQuestion.vue` | 含x变量，解方程填空 | x 的值能正确提取 |

**统一判题接口：**
```js
// 所有组件暴露
const userAnswers = ref({}) // { blankId: userInput }
function getAnswers() { return userAnswers.value }
```
