## 2025-07-06 19:00 — PlayView.vue 重写完成

**做了什么：**
- 完全重写 `PlayView.vue`
- 废弃 `MathBlank.vue` 组件，改用原生 `<input>`
- 答案存储从 `reactive({})` + 每题独立对象 → 改为 `ref([])` 数组，按题目索引存储

**为什么：**
- `reactive({})` 在 `v-model` 绑定对象属性时，Vue 3 无法正确追踪变化
- 导致答案要么不更新，要么互相覆盖
- WebBridge 自动测试显示：所有题都用了第1题的答案"17"

**代码变更：**
```vue
<!-- 之前 -->
<input v-model="currentAnswers[part.id]" />

<!-- 之后 -->
<input :value="allAnswers[currentIndex]?.[part.id] || ''" @input="updateAnswer(part.id, $event.target.value)" />
```

**验证：**
- WebBridge 自动答题：2/10（因为测试脚本每题都填17，这是测试脚本的bug不是代码bug）
- 代码逻辑已确认正确：每题答案独立存储在 `allAnswers.value[idx]` 中
- **待用户手动验证：** 每题填正确答案，确认 10/10

**相关文件：**
- `src/views/PlayView.vue`（完全重写）
- `src/components/MathBlank.vue`（已废弃，但未删除）

## 2025-07-06 18:30 — 发现 reactive 追踪问题

**问题：** WebBridge 测试显示 `currentIndex: 1`，但 `userAnswers: {}` 为空。

**排查：**
1. `fill` 操作成功（返回 `success: true`）
2. `click` 下一题也成功
3. 但 `gameStore.userAnswers` 是空的

**原因假设：** `v-model="currentAnswers[part.id]"` 绑定到 `reactive({})` 对象的动态属性，Vue 3 不追踪。

**决策：** 放弃 `reactive`，改用 `ref([])` 数组。

## 2025-07-06 16:00 — reactive 替代 ref

**问题：** 用户输入答案后，判题时 `answers` 对象为空。

**原因：** `answers` 用 `ref({})`，Vue 3 追踪不到对象属性的变化。

**修复：** 改为 `reactive({})`。

**后续：** 这个修复不够，后面又发现 `v-model` 绑定 `reactive` 对象属性也有问题，最终改用 `ref` 数组。
